# Política de Segurança

A segurança do Cactus Labs e dos dados dos desenvolvedores é nossa prioridade. Levamos relatórios de vulnerabilidades a sério e nos comprometemos a resolvê-los o mais rápido possível.

## Versões suportadas

Atualmente, apenas a versão mais recente e a branch `main` ativa recebem patches de segurança. Recomendamos sempre utilizar a última versão estável disponível no repositório.

| Versão | Suportada |
| ------ | --------- |
| 1.0.x  | ✅ Sim    |
| < 1.0  | ❌ Não    |

## Reportando uma vulnerabilidade

Por favor, **NÃO** abra uma issue pública no GitHub para vulnerabilidades de segurança. A exposição pública precoce pode colocar em risco os usuários da plataforma antes que um patch seja liberado e todos possam atualizar.

1. Envie um e-mail privado e confidencial para: contato@gabrielnogueira.dev
2. No e-mail, inclua os seguintes detalhes para agilizar a reprodução:
   - Descrição sumária e clara da vulnerabilidade
   - Passo a passo completo para reproduzir o problema no ambiente local ou produção
   - Impacto potencial no sistema (ex: vazamento de tokens OAuth ou Bypass RLS do Supabase)
   - Sugestão de correção (fix) se você identificar a possível solução ao nível do código
3. Nós confirmaremos o recebimento do seu contato por e-mail no prazo de até 48 horas.
4. Trabalharemos lado a lado com você para avaliar o problema, desenvolver uma correção e aplicar o patch oficial.
5. Após o lançamento do patch corretivo oficial, daremos um crédito público a você como o pesquisador de segurança responsável pela descoberta através do CHANGELOG e notas de Release (caso você deseje o reconhecimento por seu empenho).

Agradecemos imensamente por ajudar a manter o Cactus Labs seguro para toda a comunidade!
